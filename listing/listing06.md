Что выведет программа? Объяснить вывод программы. Рассказать про внутреннее устройство слайсов и что происходит при передачи их в качестве аргументов функции.

```go
package main

import (
	"fmt"
)

func main() {
	var s = []string{"1", "2", "3"}
	modifySlice(s)
	fmt.Println(s)
}

func modifySlice(i []string) {
	i[0] = "3"
	i = append(i, "4")
	i[1] = "5"
	i = append(i, "6")
}
```

Ответ:
```
Программа выведет [3 2 3]

Это связано с тем, что функция `modifySlice` изменяет нулевой элемент исходного слайса
на `"3"`, что видно в вызывающей функции. Однако, операции `append` могут 
привести к выделению нового массива, если вместимость слайса `i` не позволяет
добавить больше элементов. Поэтому изменения, внесенные после `append`, не 
отражаются на исходном слайсе `s` в функции `main`.

Слайсы в Go являются динамическими структурами данных, 
предоставляющими доступ к подпоследовательности элементов массива.
 Слайсы состоят из трех компонентов:

Указатель на элемент массива, с которого начинается слайс.
Длина слайса, которая указывает, сколько элементов он содержит.
Вместимость (capacity), которая говорит, как много элементов может содержать массив, 
начиная с элемента, на который указывает слайс.

Когда слайс передается в функцию, происходит копирование его структуры, 
то есть копируются указатель, длина и вместимость, но не сами элементы массива. 
В результате, изменения элементов слайса внутри функции видны вне функции, 
так как обе копии слайса ссылаются на одни и те же элементы массива. Однако, 
если слайс увеличивается за пределы его вместимости, Go выделяет новый массив, и 
новый слайс начинает ссылаться на него. В этом случае, изменения уже не будут 
отражены в исходном слайсе вне функции, так как копия слайса в функции теперь 
ссылается на другой массив.
```
